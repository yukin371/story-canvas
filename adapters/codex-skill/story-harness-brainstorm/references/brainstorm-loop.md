# Brainstorm Loop

这是项目创建前的探索轮次。

它只解决三件事：

1. 这本书到底可能卖什么
2. 它为什么能持续追更
3. 什么时候可以收束成一份 `PRD seed`

把这一轮控制在方向探索、比较和 `PRD seed` 收束。

## Round 1: 找火种

先抓最小可用火种，不追求完整世界观。

优先找：

1. 一个异常处境
2. 一个反常职业/身份
3. 一个高张力关系
4. 一个可反复启动的事件入口

如果用户只有一句模糊想法，也先把它翻译成这几个维度。

如果这一步仍然太空，转去 `question-packs.md`，用最少问题先逼出：

1. 主角异常处境
2. 追更动力
3. 单章循环

## Round 2: 扩成候选方向

至少给 `2-4` 个方向，但必须真的不同。

变化维度优先用：

1. 主角代价
2. 冲突规模
3. 单章循环
4. 读者爽点
5. 类型混搭方式

让候选方向在代价、循环、爽点或混搭方式上真正拉开差异。

如果候选方向已经多到难比较，转去 `direction-comparison.md` 做取舍。

## Round 3: 压测可连载性

对每个方向都问：

1. 主角当前最不正常的状态是什么
2. 这个状态会反复撞上什么外部压力
3. 单章最常见的触发器是什么
4. 章尾还能留下什么续追问题
5. 第一卷结束时能交付什么可检查结果

如果这五个问题答不稳，这个方向就还不够强。

## Round 4: 收束

选一个最值得追的方向，集中收束。

收束时最少要定：

1. `premise`
2. `hook`
3. `reader promise`
4. `abnormal state`
5. `scene engine`
6. `serial loop`
7. `first-volume promise`
8. `expansion board`

如果宿主支持 subagent / fresh thread，且当前两个方向都像“能写”，可加一轮挑战：

1. 把当前火种和领先方向交给无上下文 challenger
2. 让它只做两件事：
   - 质疑当前方向最大的弱点
   - 提一个更能持续追更的备选
3. 再回主线程做最终收束

这一轮不是强制，但在“两个方向都还行、难以下判”时很有价值。

如果要运行这一轮，先读 `challenger-pass.md`，按固定契约收 challenger 输出，避免 challenger 自己变成第二个主线程。

## 追问原则

优先问会改变后续产出的事。

优先级：

1. 读者为什么追
2. 主角为什么现在必须动
3. 这本书单章怎么反复成立
4. 第一卷交付是什么
5. 更大棋盘从哪里接上

低优先级：

1. 大段设定史
2. 次要角色履历
3. 暂时不影响前二十章的世界细节

## 高风险误判

优先避开这些高风险误判：

1. 只凭题材词就进入收束
2. 没做方向比较就直接认定第一想法
3. 只用“很燃”“很爽”“很治愈”描述方向，不落到异常态和单章循环
4. 先铺剧情梗概，再倒推 `premise`
