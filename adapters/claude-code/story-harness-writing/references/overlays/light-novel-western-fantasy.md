# Light-Novel Western-Fantasy Overlay

适用定位：

1. `primaryGenre=fantasy`
2. `subGenre=western-fantasy`
3. `styleTags` 包含 `light-novel`
4. 常伴随 `short-fiction`
5. 目标读者接近 `ya-reader`、`light-novel-reader`

证据基础：

1. `projects/demo-light-novel-short/README.md`
2. `projects/demo-light-novel-short/MODULE.md`
3. `projects/demo-light-novel-short/project.yaml`

## 当前高置信写法

1. 开场优先给角色互动、异常小事故或轻巧钩子，不要先灌长段设定。
2. 西幻元素要服务可爱的冒险感、搭档感和小型奇观，而不是沉重史诗腔。
3. 每章最好有一个明确兑现点，例如：
   - 搭档关系推进
   - 小危机解决
   - 魔法/生物设定的趣味落地
   - 角色个性反差带来的小回报
4. 语言气质应轻快、清楚、带一点俏皮，但不能滑成纯段子堆积。
5. 章节结尾更适合留下：
   - 下一段小冒险入口
   - 新搭档默契变化
   - 新奇生物/道具的麻烦
   - 看似轻松但会继续扩大的误会或任务

## 节奏建议

1. 首章先让读者喜欢角色和组合，再逐步扩世界。
2. 若是短篇，不要把篇幅耗在背景史；优先保证三章内能完成一次完整小冒险。
3. 冲突可以不沉重，但不能没有成本；至少要让角色承担选择、失误或关系变化。
4. 若同时带 `short-fiction`，每章都要朝终点收束，避免 web-serial 式无尽铺垫。

## 常见误区

1. 西幻名词很多，但角色化学反应很弱。
2. 想写轻小说气质，却只剩口癖和卖萌。
3. 设定说明压过剧情，导致轻快感被讲解拖垮。
4. 短篇结构里仍按长连载写法反复预埋，不肯及时兑现。

## 与普适层的分工

这层补的是：

1. 西幻轻小说的语气与互动期待
2. 短弧线的小冒险兑现节奏
3. 设定轻量化与角色化的落地方式

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
