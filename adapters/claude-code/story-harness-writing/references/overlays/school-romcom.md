# School-Romcom Overlay

适用定位：

1. `primaryGenre=romance`
2. `subGenre=school-romcom`
3. 常伴随 `campus`、`light-novel`
4. 可兼容 `dependency-romcom`、`club-life`
5. 目标读者接近 `light-novel-reader`、`campus-romcom-reader`

证据基础：

1. `noval-data/analysis-workbench/synthesis/genre-overlays/school-romcom.md`
2. `noval-data/review-index.md`

## 当前高置信写法

1. 主角的关系目标要尽快清楚，不要长期只靠热闹日常撑章。
2. 日常场景本身也要持续交付关系变化、对象选择变化和情绪波动。
3. 对白要同时承担人物辨识、情绪推进和轻喜剧节奏，不能只剩俏皮话。
4. 若带病娇或高依存关系，边界推进要靠具体越界行为、照顾、独占或失控，而不是口头宣示。
5. 若有社团、乐队或放学后固定聚点，它最好成为稳定的关系发生器，而不是装饰背景。

## 节奏建议

1. 首章先立住主角的关系目标和第一轮高频互动场。
2. 前几章至少要出现一次明确的关系增量：靠近、误会、试探、吃醋或共同秘密。
3. 第一卷最好交付初步对象盘面和一个可持续复用的固定互动舞台。
4. 若走依存支线，第一卷最好出现一次从轻喜剧贴近转入真实关系危险的节点。

## 常见误区

1. 只有热闹日常，没有关系增量。
2. 主角目标不够直接，推进长期被动。
3. 配角很多，但不服务对象选择。
4. 固定社交场存在感很高，却不持续产出关系变化。
5. 依存感只靠台词喊出来，没有具体越界和代价。

## 与普适层的分工

这层补的是：

1. 校园恋爱喜剧的关系推进密度
2. 固定社交场的使用方式
3. 依存或高黏性关系的具体推进手法

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
