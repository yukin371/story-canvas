# Xuanhuan Web-Serial Overlay

适用定位：

1. `primaryGenre=fantasy`
2. `subGenre=xuanhuan`
3. `styleTags` 包含 `web-serial`
4. 目标读者接近 `qidian-reader`、`progression-fantasy-reader`

证据基础：

1. `projects/demo-xuanhuan-short/README.md`
2. `projects/demo-xuanhuan-short/MODULE.md`
3. `projects/demo-xuanhuan-short/project.yaml`

## 当前高置信写法

1. 开篇要尽快立住压制处境、资源稀缺和反压冲动，不要先做大段体系说明。
2. 境界、门槛、代价要尽早给到最低可理解版本，让读者知道主角差在哪、要怎么追。
3. 章节推进优先看：
   - 压制升级
   - 反压尝试
   - 体系信息落地
   - 小型能力兑现
4. 场景里最好有明确层级差：人压人、门规压人、资源压人、境界压人。
5. 章节结尾更适合留下：
   - 新门槛
   - 新敌意
   - 新资源机会
   - 小升级后的更高代价

## 节奏建议

1. 首章先让读者看懂主角为什么弱、为什么还不能退。
2. 前几章至少要有一次“不是空喊变强，而是真有局部反制或资格变化”的兑现。
3. 若是短篇或首卷开端，不要把升级写成纯设定课；至少要绑定具体冲突。

## 常见误区

1. 只有玄幻名词，没有可感知压制。
2. 境界体系讲很多，但当前章没有实际用途。
3. 主角长期挨打却没有代价差异或认知推进。
4. 把“变强欲望”写成口号，而不是场景中的刚需。

## 与普适层的分工

这层补的是：

1. 升级阅读期待
2. 压制与反压结构
3. 境界与资源的场景化落地

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
