# Historical-Power Overlay

适用定位：

1. `primaryGenre=historical`
2. `subGenre=historical-power`
3. 常伴随 `war`、`governance`、`state-building`
4. 可兼容 `industrial-pivot`、`chaos-rising`
5. 目标读者接近 `historical-web-serial-reader`

证据基础：

1. `noval-data/archetypes/historical-power/README.md`
2. `noval-data/analysis-workbench/synthesis/genre-overlays/historical-power.md`

## 当前高置信写法

1. 开篇先把时代危机压到一个具体缺口上，例如军工、钱粮、筑城、整编或人口组织，不要只讲大历史。
2. 主角的现代认知或特殊优势，要尽快落成真实项目成果，而不是停在“他懂很多”。
3. 项目推进必须带来战局、权力或合作关系的可感知变化。
4. 若走乱世经营线，人口、粮食、地盘和组织能力都要逐步可见，不要只剩杀伐。
5. 若带系统或奖励，它只能加速落地，不能替代真实实施路径和组织建设。

## 节奏建议

1. 首章先写最紧迫的现实缺口，以及主角为什么能切进去。
2. 前几章至少要完成一次“小项目落地 -> 更高层任务或代价抬升”。
3. 第一卷更稳的交付，是主角从地方项目操盘者或底层求生者，升级为国家级危机或乱世牌桌里的关键变量。
4. 卷级兑现最好锚到具体战役、装备、城防或政商节点。

## 常见误区

1. 时代危机只是背景板，没有压到具体项目。
2. 主角总在判断正确，却缺少真正落地成果。
3. 项目成功只有赚钱反馈，没有权力后果和战局变化。
4. 经营写成流水账，时代高压感逐渐消失。
5. 建设线只靠系统发奖励，没有真实组织能力。

## 与普适层的分工

这层补的是：

1. 历史经营与时代危机的绑定方式
2. 项目成功如何转成战局与权力变化
3. 乱世扩盘与秩序重建的章节推进器

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
