# ACG-Fanfic-Action Overlay

适用定位：

1. `primaryGenre=fanfic`
2. `subGenre=acg-fanfic-action`
3. 常伴随 `familiar-ip`、`external-power-system`
4. 可兼容 `canonical-reroute`、`modern-academy`、`chatgroup-route`、`multiworld-route`
5. 范围护栏：
   - 适合“熟悉 IP + 外来体系改线 + 动作牌桌改写”变体
   - 暂不覆盖所有 ACG 同人写法

证据基础：

1. `noval-data/analysis-workbench/synthesis/genre-overlays/acg-fanfic-action.md`
2. `noval-data/review-index.md`

## 当前高置信写法

1. 熟悉 IP 场景和角色要快速带人入戏，但不能只靠认脸维持推进。
2. 外来力量体系不仅是爽点，还要持续改写原作秩序和人物站位。
3. 主角要从原作世界民间变量，逐步变成真正的牌桌变量。
4. 若切口是原著失败者、校园主线前置、都市综漫案件、聊天群或多世界任务，都要尽快写清本轮入口和资源规则。
5. 每次出手后都要补结构后果，而不是只停在打赢。

## 节奏建议

1. 首章先选一个读者最熟悉的场景、组织或日常入口。
2. 前几章至少要让主角改掉一段原作默认秩序、败局或关系走向。
3. 第一卷更稳的交付，是主角完成从民间变量到正式牌桌关键人的第一次跃迁。
4. 若走聊天群、多世界或圣女代行线，第一卷最好至少交付一次规则被验证后的完整闭环。

## 常见误区

1. 过度依赖读者认角色，原创推进发空。
2. 外挂只负责压制战斗，没有组织后果和秩序后果。
3. 熟悉角色出场很多，却没有谁真正因主角而改线。
4. 跨 IP 设定只剩彩蛋和梗，没有形成独特世界解释。
5. 多世界、群聊或公共身份只剩皮肤，没有清楚资源流转和回收机制。

## 与普适层的分工

这层补的是：

1. 熟悉 IP 场景的切入方式
2. 外来体系如何改写原作秩序
3. 动作同人的结构后果与改线闭环

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
