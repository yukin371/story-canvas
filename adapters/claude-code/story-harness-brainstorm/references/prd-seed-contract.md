# PRD Seed Contract

当用户要从探索进入收束时，再读取本文件。

`PRD seed` 不是正式 `PRD.md`，但应该足够让后续 `init`、写作 skill 或人工继续推进。

## Required Fields

最少输出：

1. 主类型
2. 目标平台或传播场景
3. 目标读者
4. 一句话 `premise`
5. `hook line`
6. 主角异常状态
7. 核心反差
8. 读者承诺
9. 场景引擎
10. 连载循环
11. 第一卷承诺
12. 扩张棋盘

## Output Shape

用这个格式：

```markdown
## PRD Seed

**题材 / 平台 / 人群**
- 主类型：
- 目标平台：
- 目标读者：

**故事承诺**

- hook：
- 读者承诺：

**主角与冲突**
- 主角异常状态：

- 当前外部压力：

**连载模型**
- 场景引擎：
- 连载循环：
- 第一卷承诺：
- 扩张棋盘：

- 1.

```

## Quality Bar

合格的 `PRD seed` 应满足：

1. 不靠空泛题材词支撑
2. 能解释为什么读者会继续追
3. 能解释单章怎么重复成立
4. 第一卷交付可被检查
5. 还没确定的地方被明确列成待确认项

## Handoff

如果用户接下来要进入项目执行，建议的下一步是：

1. 用 `story-harness-writing`
2. 运行 `story-canvas init`
3. 把这里的关键信息写进真实 `PRD.md` 和项目定位字段
