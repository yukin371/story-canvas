# 卷级自审门与人工审查时机

## 1. 背景

当前仓库已经能较稳定跑通单章写作闭环：

- `outline check`
- `chapter analyze`
- `chapter suggest`
- `review apply`
- `projection apply`
- `context refresh`
- `review chapter`
- `review scene`

但这还不等于已经完成“适合人工终审的故事闭环”。

原因很直接：

- 单章通过，不代表整卷阅读体验成立。
- 章内分数正常，不代表章间承接自然。
- review 能指出局部弱项，不代表能自动发现整卷层面的角色弧线、冲突节奏和兑现问题。

因此，当前实验阶段必须把“卷级 AI 自审门”明确写入仓库流程，而不是继续靠口头约束记忆。

## 2. 目标

明确三件事：

1. `章节闭环` 和 `卷级闭环` 不是一回事。
2. 人工审查的默认进入时机，应在一个完整卷/小故事单元完成并经过 AI 卷级自审之后。
3. 每次人工审查发现的问题，都要追问：
   - 为什么生成时没做好
   - 为什么 AI 自审时没发现
   - 为什么 CLI / workflow / review 没揭露

## 3. 非目标

- 不是本轮就实现新的 CLI 命令或 schema 变更
- 不是把人工审查替换成 AI
- 不是要求每一章都做一次重量级整卷复盘
- 不是把所有问题都硬编码成规则

## 4. 两层闭环

### 4.1 章节闭环

章节闭环的目标是保证：

- 本章具备最小前置约束
- 本章正文经过基础检测
- 本章完成一轮自审和修正
- 本章可以安全进入下一章写作

章节闭环完成后，允许：

- 继续写下一章
- 刷新 context
- 继续积累卷内状态

章节闭环完成后，不自动意味着：

- 可以直接进入人工终审
- 可以断言整卷逻辑、承接、角色弧线已经成立

### 4.2 卷级闭环

卷级闭环的目标是保证：

- 一个完整卷或一个明确的小故事单元已经写完
- AI 先从整卷视角复查连续阅读体验
- AI 先做一轮缺陷归因和修正
- 再把相对收敛的版本交给人工审查

默认流程应为：

1. 卷内所有目标章节先完成章节闭环
2. 汇总整卷上下文、评审结果和正文
3. 执行一轮卷级 AI 自审
4. 根据卷级自审结果修改正文或局部结构
5. 对受影响章节复检
6. 再进入人工审查

## 5. 卷级 AI 自审门的必审维度

至少覆盖以下方向：

- 设定遵循是否稳定
- 世界与背景逻辑是否连续
- 章与章之间是否自然承接
- 人物动机、反应、关系推进是否稳定
- 冲突是否逐步升级，而不是平推细纲事项
- 高潮、反转、兑现和卷尾钩子是否成立
- 行文是否出现明显 AI 味、方案文档腔、总结腔
- 是否存在“为了贴细纲而牺牲自然发展”的痕迹

其中，“章间衔接”必须单独列项，不能再混进笼统的 review 分数里。

## 6. 卷级 AI 自审的最小产物

在当前实验阶段，每次卷级自审至少要形成一份固定审查包，内容包括：

1. 卷范围
2. 当前卷的目标承诺
3. 关键角色与线程的卷内推进摘要
4. CLI 已发现的问题摘要
5. AI 新发现的问题清单
6. 每个问题的归因分类
7. 修正动作
8. 复检结论
9. 剩余风险
10. 是否允许进入人工审查

推荐问题归因分类固定为：

- `generation_miss`
- `self_review_miss`
- `tooling_miss`
- `false_positive`
- `accepted_risk`

这样后续才能判断，问题究竟该回灌到：

- prompt / workflow 约束
- AI 自审口径
- style / consistency / review / doctor / workflow
- 还是继续保留为人工判断

## 7. 人工审查的默认时机

默认不在“某一章刚写完”时进入人工终审。

默认进入人工审查的时机应满足：

1. 一个完整卷或小故事单元已经写完
2. 卷内章节已完成章节闭环
3. AI 已完成一轮卷级自审
4. 关键缺陷已至少修正一轮
5. 受影响章节已完成复检

只有在以下情况下，才允许提前人工介入：

- 用户明确要求中途抽查
- 当前问题疑似涉及根本性方向错误，继续写会放大成本
- 工具或 AI 已明显失真，继续自动迭代价值不高

## 8. 当前实验阶段的执行口径

在 dedicated 工具能力补齐前，当前仓库内的正式口径是：

- `workflow` 命令当前仍主要表达单章 gate
- `creative-workflow` guide 负责定义“章节闭环 -> 卷级闭环 -> 人工审查”的人工执行流程
- `roadmap` 负责把卷级自审门写成 `v1.0` 发布前必须证明的能力

也就是说：

- 现在可以先用文档与固定产物约束流程
- 但这不是终态
- 后续应把这套卷级自审门直接收敛进工具，而不是长期靠人手执行

## 9. 后续工具化目标

后续应优先考虑把以下能力产品化：

1. `workflow` 增加卷级 gate，而不只停留在单章 gate
2. 提供卷级 review packet 导出，而不是逼着操作者手翻 YAML / review 文件
3. 把“是否允许进入人工审查”变成显式 gate 结果
4. 把问题归因分类固化到 review / workflow 产物
5. 把“生成漏做 / 自审漏看 / 工具漏报”的回灌路径做成可追踪状态

## 10. 验证口径

后续真实写作样例应至少验证：

1. 单章闭环和卷级闭环是否被明确区分
2. AI 是否能在人工审查前先找出一批整卷问题
3. 人工审查发现的问题，是否能被稳定归类到 `generation/self-review/tooling`
4. 卷级自审是否确实降低人工审查负担，而不是再增加一轮形式主义
5. 操作者是否还需要频繁手翻 YAML 和 review 文件才能理解当前状态

## 11. 当前结论

对 `v1.0` 而言，真正需要证明的不是“单章可以跑出一份正文”，而是：

- 单章能闭环
- 整卷能自审
- 人工审查发生在正确时机
- 人工发现的问题能反过来改进生成、自审和工具

这应被视为当前实验阶段的正式流程要求，以及后续工具化收敛目标。
