# 无来源姓名揭露检测计划

> 日期: 2026-04-26
> 状态: 进行中
> 范围: `services/consistency_engine.py` / `services/story_review.py` / 对应 smoke tests

## 1. 背景

当前项目已经能检查设定冲突、题材语域失真、叙事支架复用与已包裹实体建档情况，但仍缺一类会直接破坏阅读自然度的问题：

1. 角色尚未通过合理渠道被 POV 或读者获知姓名。
2. 正文却突然直接给出名字，尤其是“特征描写块 -> 单独一句姓名”的写法，会显得像作者或模型跳过了信息获取过程。
3. 这类问题本质上不是单纯风格词汇问题，而是叙事认知边界问题，更适合进入一致性校验层。

## 2. 目标

1. 在 `consistency_engine.py` 增加高置信“姓名无来源提前揭露”检测。
2. 只做轻量启发式，不引入 NLP 依赖，不尝试判定所有首次登场命名。
3. 让该信号进入 `review chapter` / `review scene` 的风险提示和优先动作。

## 3. 启发式边界

- 重点抓以下高置信模式：
  - 前文先出现匿名角色特征块，如身份、兵器、气息、外貌片段
  - 随后单独一小段直接给出 2-4 字中文姓名
  - 附近没有“自报名号 / 他人称呼 / POV 听见名字 / 明确得知”之类引介来源
- 非目标：
  - 不做完整命名实体识别
  - 不强行判断所有旁白直呼姓名都是错误
  - 不处理标题、名单、已明确介绍后的正常称呼

## 4. 验证方式

- `python -m unittest tests.smoke.test_consistency_engine tests.smoke.test_review_chapter`
- 如无异常，再跑相关 smoke 组合与全量回归
