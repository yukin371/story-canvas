# 细纲约束与人物自然发展冲突处理

> 日期: 2026-04-26
> 状态: 草稿
> 关联当前入口: `docs/roadmap.md`
> 关联文档:
> - `docs/plans/2026-04-26-writing-quality-loop.md`

## 1. 背景

长篇创作里，细纲是必要约束，但不能把它误当成逐句执行脚本。

真实问题是：

- 若完全硬贴细纲，正文容易变成“提纲执行稿”
- 若完全放开让人物自由跑，长线承诺、伏笔窗口、卷级推进又会失控

因此需要的不是二选一，而是明确：

1. 哪些约束必须守
2. 哪些计划可以调整
3. 偏离后如何回收

## 2. 核心结论

细纲不是最高优先级真相源。

当“细纲安排”和“人物在当前设定、关系、压力、后果下的合理反应”发生冲突时，处理原则应是：

### 2.1 优先守住硬约束

硬约束包括：

- 世界规则
- 已发生的章节结果
- 当前角色状态与已建立动机
- 关键故事承诺与卷级主线目标

### 2.2 允许调整软计划

软计划包括：

- 某个 scene 的具体执行方式
- 某场冲突由谁先开口、谁先行动
- 某条信息在本章前半还是后半揭露
- 细纲里某一步是否需要补桥接段才能成立

### 2.3 偏离不是放飞，而是要回收

如果人物自然发展导致细纲不能原样执行，不应直接视为失败，而应进入：

- 标记偏差
- 判断是否仍服务主线与章节目标
- 必要时重写 scenePlans / beats / bridging
- 再走 review / style / consistency 复检

## 3. 约束优先级

建议把写作约束分成 4 层：

### Level 1: 世界与事实硬约束

- `worldbook`
- 已确认设定
- 已发生事件后果
- 已建立的能力边界、身份边界、时间线边界

这一层不能为了贴细纲而违背。

### Level 2: 角色与关系硬约束

- 当前角色状态
- 当前情绪与压力
- 已建立的人物弧线方向
- 当前关系张力和已发生的关键互动后果

这一层也不应被机械细纲覆盖。

### Level 3: 章节与卷级目标约束

- 本章必须交出的推进
- 本章对上一章的承接义务
- 本卷当前应推进的主线、伏笔、情绪目标

这一层通常要守，但允许通过“改写路径”而不是“原样照抄细纲”来完成。

### Level 4: 细纲与 scenePlans 执行路径

- 具体 beat 顺序
- 场景调度方式
- 对话/冲突触发顺序
- 信息揭露节拍

这一层默认是可调整的。

也就是说：

- 细纲是实现方案
- 不是最高法律

## 4. 何时允许人物自然发展压过细纲

以下情况，应优先尊重人物与故事现场逻辑：

1. 细纲安排会让人物明显 OOC。
2. 细纲安排会让上一章刚建立的情绪/关系结果立刻失效。
3. 细纲安排会迫使角色跳过合理反应，直接服务剧情推进。
4. 细纲安排会让章节承接断裂，只剩“事项完成”而没有自然过渡。

这时正确做法不是硬拉人物回线，而是：

- 先承认现场逻辑更强
- 再调整 scenePlan 或 beat path
- 同时检查是否仍完成本章目标

## 5. 何时不应允许“自然发展”当借口

以下情况，不能简单说“人物自然发展”：

1. 实际上只是模型跑偏，没有明确人物因果依据。
2. 偏离后丢失本章关键推进，导致长线承诺失踪。
3. 偏离后引入新的设定、关系或动机，但没有补足解释。
4. 所谓“自然”只是拖节奏、绕开难写场景、回避关键冲突。

这类情况本质上不是“人物更合理”，而是生成质量不足。

## 6. 建议工作流

### 6.1 写前

在开写前，不只要给细纲，还要给：

- 上章结果
- 本章承接点
- 本章必须交出的推进
- 当前角色状态
- 当前不允许违背的关系/规则/情绪边界

同时把细纲内容分成两类：

- `must_keep`
- `flexible_path`

### 6.2 写中

如果人物自然发展与细纲冲突，先问 3 个问题：

1. 这个偏离是否更符合角色当前状态与世界规则？
2. 这个偏离是否仍然服务本章和本卷目标？
3. 这个偏离是否需要补桥接段、补 scene、改 beats？

若前两项成立，就允许局部改 path；否则应拉回。

### 6.3 写后

成稿后人工审查和 CLI 应重点问：

1. 这一章是不是只是把细纲写成了正文？
2. 人物行为是否像真实反应，而不是剧情工具？
3. 若已偏离细纲，是否有清楚的回收与更新？
4. 偏离后，上一章承接、本章推进、下一章起点是否更顺了还是更乱了？

## 7. 对系统能力的要求

后续产品化时，建议系统显式支持：

### 7.1 细纲项分类

- `required`
- `preferred`
- `optional`

而不是默认所有 beat 都必须原样发生。

### 7.2 偏差记录

当正文没有按原 scenePlan 走时，应能记录：

- 偏离点
- 原计划
- 实际走向
- 偏离原因
- 是否需要同步更新 outline / scenePlans

### 7.3 review 口径

`review` 不应只检查“有没有覆盖细纲内容”，还应检查：

- 是否违背角色现场逻辑
- 是否因为硬贴细纲而损害衔接、人物、自然性

## 8. 当前结论

对长篇小说而言，更合理的口径不是：

- “故事必须完全遵循细纲”

而是：

- “故事必须遵循世界规则、既成结果、角色逻辑和长线承诺；细纲是实现这些目标的首选方案，但不是不可修改的绝对脚本。”

这样既能避免正文机械执行细纲，又不会把“自然发展”变成失控跑偏的借口。
