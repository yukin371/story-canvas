# 功能改进阶段划分与并行实施建议

> 日期: 2026-04-27
> 状态: 草稿
> 关联当前入口: `docs/roadmap.md`
> 关联 PRD: `docs/plans/2026-04-27-writing-quality-workflow-prd.md`
> 关联收敛文档: `docs/plans/2026-04-27-post-review-convergence-plan.md`

## 1. 目的

把后审查收敛出的功能改进拆成可执行阶段，明确：

- 先做什么
- 后做什么
- 哪些工作包能并行
- 哪些工作必须串行
- 哪些适合后续交给代理并行推进

## 2. 阶段总览

### Phase 0: 规则收敛与审查口径固定

目标：

- 先统一“AI 应该怎么写、怎么自审、什么算一卷完成”的规则口径。

交付：

- 第一版写作规则包
- 卷级评分维度表
- AI 自审报告模板
- 首卷引入 / 卷级闭环 / 风格禁区的规则分类

为什么先做：

- 如果规则不先定，后面的工具和检测都会继续漂移。

### Phase 1: CLI 基础工具化

目标：

- 先把最影响主工作流的 YAML 相关动作收回 repo-native CLI。

交付：

- 新章创建
- 状态浏览
- scenePlans 边界回填
- 卷级导出
- 卷级审查包导出

为什么第二个做：

- 没有这些能力，AI 仍会频繁直接改协议文件，产品闭环不成立。

### Phase 2: 故事钩子进入主链

目标：

- 让实体、伏笔、设定、势力等真相层真正进入写作与审查链。

交付：

- 实体 mention 扫描、包裹建议、补档建议、章节摘要注入
- 伏笔到窗提醒与未回收审查
- 核心概念 onboarding 审查
- 势力层级 / 地域扩张 / 战力阶段审查

为什么第三个做：

- 这一步依赖基础工具和规则口径，但完成后才能真正减少“AI 靠自己记忆写”的风险。

### Phase 3: 卷级闭环与风格增强

目标：

- 补齐卷级 gate、卷级评分、自审修复循环与更强的 AI 风格检测。

交付：

- 卷级闭环 gate
- 爽点引擎 / 核心体验 / 目标读者结构化审查
- 卷级结构模板与结构完成度审查
- 高频支架句风格习惯检测增强
- optional dependency 风格增强路径

为什么最后做：

- 这一步最依赖前面三阶段的规则、工具和故事钩子基础。

## 3. 各阶段工作包

### Phase 0 工作包

#### P0-A: 写作规则包第一版

范围：

- 开篇规则
- 首卷规则
- POV 规则
- 风格规则
- 角色 / 反派规则
- 卷级闭环规则

适合并行：

- 可以拆成多份规则清单草稿并行整理，最后统一收敛。

#### P0-B: 卷级评分与 AI 自审模板

范围：

- 评分维度
- 缺陷归因格式
- 修复循环格式

依赖：

- 依赖 P0-A 的规则口径

### Phase 1 工作包

#### P1-A: 新章创建与结构同步

范围：

- 新建章节
- 同步 outline direction / beats / scenePlans
- 推进 `activeChapterId`

当前进度：

- 已完成最小可用版本：`chapter create`

#### P1-B: scenePlans 边界回填与合法性校验

范围：

- 边界重算
- off-by-one 校验
- review 前预检

当前进度：

- 已完成最小可用版本：`outline scene-sync`

#### P1-C: 项目 / 章节 / 审查状态浏览

范围：

- 项目状态
- 当前卷/当前章状态
- 最近 review / style / consistency 状态

当前进度：

- 已完成最小可用版本：`status`

#### P1-D: 卷级导出与卷级审查包

范围：

- `第一卷.md` / `第一卷.txt`
- 卷级结构稿
- AI 自审结论聚合

当前进度：

- 已完成最小可用版本：`export --volume-id` / 卷级 `review-packet`

适合并行：

- `P1-A` 与 `P1-C` 可并行
- `P1-B` 与 `P1-D` 可并行

### Phase 2 工作包

#### P2-A: 实体闭环

范围：

- 已知实体自动包裹建议
- 缺失实体补档提示
- 同章去重反馈
- 当前章实体设定摘要注入

当前进度：

- 已完成最小可用版本：`entity mention-check`
- 已补 repo-native 维护入口：`entity add` / `entity state-update` / `entity mention-adopt`
- 已补章节级 action preview / apply：`entity mention-plan` / `entity mention-apply`

#### P2-B: 伏笔闭环

范围：

- 长短期伏笔分类
- due window 提醒
- 未回收 warning / gate

当前进度：

- 已完成最小可用版本：`foreshadow check`

#### P2-C: 核心概念与世界尺度审查

范围：

- 核心概念最晚解释时点
- 制度压迫成立条件
- 势力层级 / 地域扩张 / 战力阶段

当前进度：

- 已完成最小可用版本：`world check`
- 当前输出 `storyTemplate.modulePolicy` 世界模块缺口、当前章/当前卷相关世界项上下文、早期 onboarding 缺口、势力建档薄弱项，以及复用一致性引擎的高风险任务/突破链冲突
- 已补 repo-native worldbook 维护入口：`world list` / `world add` / `world mention-adopt` / `world progression-add` / `world progression-stage-add`

适合并行：

- `P2-A`、`P2-B`、`P2-C` 可并行推进，但最终都要接回 `review / context / doctor`

### Phase 3 工作包

#### P3-A: 卷级闭环 gate

范围：

- 卷主问题收束
- 阶段性交付
- 卷尾余波与完成感平衡

#### P3-B: 爽点引擎 / 商业承诺审查

范围：

- 目标读者
- 核心体验
- 主爽点引擎
- 压抑 / 爆发分布
- 阶段性兑现节点

#### P3-C: 卷级结构模板

范围：

- 三幕式
- 五幕式
- 引入卷结构检查表

#### P3-D: 风格增强

范围：

- 高频支架句习惯检测
- 抽象递进检测
- 角色口吻归属
- optional dependency 增强路径

适合并行：

- `P3-B`、`P3-C`、`P3-D` 可并行
- `P3-A` 需在其他几包有最小可用结果后再统一接 gate

## 4. 依赖关系

### 必须先完成

- `Phase 0` 必须先于全部后续阶段

原因：

- 不先固定规则，后面的功能很容易边做边改口径。

### 强依赖

- `Phase 1` 是 `Phase 2` 的强依赖

原因：

- 如果没有 repo-native CLI 能力，实体/伏笔/设定闭环仍会退回 AI 直接读改协议文件。

### 中依赖

- `Phase 2` 是 `Phase 3` 的中依赖

原因：

- 卷级 gate 和卷级评分必须消费实体、伏笔、设定和世界尺度信号，才能不空转。

## 5. 适合代理并行的工作

后续若用代理并行，优先拆给代理的应是“边界清晰、写集分明”的工作包。

### 适合 explorer 并行

- 规则清单对齐
- 样例句库整理
- 评分维度归并
- 文档口径核对

### 适合 worker 并行

- `P1-A` 新章创建与结构同步
- `P1-B` scenePlans 边界回填
- `P1-C` 状态浏览
- `P1-D` 卷级导出 / 审查包
- `P2-A` 实体闭环
- `P2-B` 伏笔闭环
- `P2-C` 核心概念 / 世界尺度审查
- `P3-D` 风格增强

### 不适合完全外包给代理的工作

- PRD 最终收敛
- 规则包最终定稿
- 阶段边界裁剪
- 卷级 gate 的最终判断标准

原因：

- 这些属于产品级主判断，必须由主线程统一收口，避免多代理各自定义真相源。

## 6. 推荐并行切片方式

### 第一批并行

- 写作规则包整理
- 卷级评分与自审模板整理

### 第二批并行

- 新章创建 / 结构同步
- 边界回填 / 合法性校验
- 状态浏览
- 卷级导出 / 审查包

### 第三批并行

- 实体闭环
- 伏笔闭环
- 核心概念 / 世界尺度审查

### 第四批并行

- 爽点与商业承诺审查
- 卷级结构模板
- AI 风格增强

## 7. 每阶段验收标准

### Phase 0 验收

- 第一版规则包可直接作为 AI 写前约束输入
- 卷级评分与 AI 自审模板固定下来

### Phase 1 验收

- AI 推进正文时，不再频繁直接改 YAML
- 卷级导出与审查包可独立交给人工阅读

### Phase 2 验收

- 实体、伏笔、设定不再只是存档，而能进入当前章/当前卷审查链

### Phase 3 验收

- 系统能明确判断“有高潮但未闭环”与“卷已闭环”
- 系统能对 AI 风格习惯问题给出更可用的修文指向

## 8. 当前建议执行顺序

1. 先完成 `Phase 0`
2. 再启动 `Phase 1`
3. `Phase 1` 过最小可用后并行推进 `Phase 2`
4. 最后用 `Phase 3` 收卷级闭环与风格增强

原因：

- 先定规则
- 再补工具
- 再让故事真相层进入主链
- 最后补更强的卷级 gate 和风格增强
