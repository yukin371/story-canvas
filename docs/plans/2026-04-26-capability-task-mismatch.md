# 能力层级与任务风险不匹配检测计划

> 日期: 2026-04-26
> 状态: 进行中
> 范围: `services/consistency_engine.py` / `services/story_review.py` / 对应 smoke tests

## 1. 背景

人工审查发现一类常见设定失真：

1. 角色当前修为或公开实力明显偏低。
2. 正文却让其直接参与高风险任务，例如秘境探索、险地深入、宗门高危试炼。
3. 文中又没有补充世界观例外、安全边界、护持条件或任务门槛说明。

这类问题会削弱修炼体系、宗门制度和危险等级的可信度。

## 2. 目标

1. 在 `consistency_engine.py` 增加高置信软校验，识别“低修为角色参与高风险任务但缺少例外说明”的问题。
2. 不做完整战力系统推导，只覆盖最明显的玄幻/仙侠语境。
3. 让信号进入 chapter / scene review 的优先动作和风险提示。

## 3. 启发式边界

- 只抓以下高置信组合：
  - 角色卡里存在低修为/低实力标记
  - 正文附近明确出现该角色姓名
  - 同段或近邻段落出现高风险任务词
  - 附近缺少“外围 / 长老带队 / 护符 / 只限低阶 / 禁制压制”等安全边界说明
- 非目标：
  - 不判断复杂越阶战斗是否一定不合理
  - 不覆盖都市异能、现代修仙等所有混合题材
  - 不替代作者对世界规则的创造性例外设定

## 4. 验证方式

- `python -m unittest tests.smoke.test_consistency_engine tests.smoke.test_review_chapter`
- 通过后再跑全量 `discover`
