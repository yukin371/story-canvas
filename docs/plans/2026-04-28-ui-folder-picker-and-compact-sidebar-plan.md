# UI 目录选择与紧凑侧栏计划

> 创建日期: 2026-04-28
> 状态: 进行中
> 关联当前入口: `docs/roadmap.md`

## 一、背景

当前工作台的“导入项目”仍依赖手输绝对路径，交互上不是真正的选目录；同时插画侧栏参数区仍偏松散，和目标中的 VS Code / TDesign 风格不一致。

## 二、目标

1. 为本地 UI API 增加真实目录选择接口
2. 将项目导入改为“路径输入 + 选择目录 + 导入项目”
3. 压缩插画侧栏参数区，改为标签左侧、控件右侧的紧凑单行布局

## 三、适用规则

- UI 不是新的真相源
- `projects/` 自动发现仍是默认项目源
- 外部导入项目只允许写入工作台私有注册
- 本地文件夹选择必须通过本地 UI API 桥接，不伪装成浏览器文件上传

## 四、实施项

### 4.1 本地 API

- 在 `scripts/story_canvas_ui_api.py` 增加目录选择 helper
- 暴露 `/api/dialogs/select-folder` 接口
- 返回选中目录绝对路径；取消时明确返回错误

### 4.2 前端工作台

- 在 `ui/src/api/storyCanvas.ts` 增加目录选择请求封装
- 在 `ui/src/views/WorkbenchView.vue` 的项目导入面板接入目录选择按钮
- 保留手输路径作为 fallback，但默认引导用户使用选择目录

### 4.3 样式压缩

- 收紧插画侧栏参数项的纵向间距
- 将参数表单改为 label / control 同行布局
- 在窄屏下保留回退布局，避免可用性回退

## 五、验证

- `python -m py_compile scripts/story_canvas_ui_api.py`
- `npm run build`

## 六、风险

- 本地目录选择依赖桌面环境；需提供清晰错误信息与手输路径 fallback
- 侧栏布局压缩后，需避免小屏时控件挤压到不可操作
