# 写作审查缺口记录

> 历史状态: 2026-04-29 起降级为 `historical-source`
> 说明: 本文继续保留人工审查回溯价值，但不再作为长期活跃痛点入口。后续仍会增加 AI 实施负担的条目，应统一收敛到 `docs/tracking/ai-friction-tracker.md`。

> 日期: 2026-04-26
> 状态: Historical source
> 用途: 记录人工审查中发现、但系统尚未稳定覆盖的写作问题，作为后续 review / consistency / export 能力补强输入

## 1. 旧导出串章标题

- 现象:
  - 上一章文末会串入下一章标题。
  - 示例:
    - `但脑海中反复浮现的，始终是那道银色的剑光——以及剑光中那一丝若有若无的归墟气息。第五章 意外之人`
- 影响:
  - 破坏章节边界
  - 导致阅读节奏和断章钩子失真
- 初步归类:
  - `export` / 章节拼接边界问题

## 2. 修为阶段与任务风险不匹配

- 现象:
  - 主角加入宗门后仍处于练气期甚至未入练气，却被直接派去秘境探索。
  - 以常规仙侠/玄幻语境看，低修为阶段通常缺少法术与自保手段，难以合理承担高风险环境探索任务。
  - 当前正文没有充分交代世界观例外，例如：
    - 该秘境本身被限定为低阶弟子试炼场
    - 宗门提供护符、带队长老、固定安全边界
    - 本世界练气期已有基础术法或特殊保命机制
- 影响:
  - 容易让读者感到设定失真
  - 会削弱宗门制度、修炼体系与危险等级的可信度
- 初步归类:
  - 世界规则显性化不足
  - 角色能力与任务门槛不匹配
  - 任务分配合理性缺少前置交代
- 后续建议检查方向:
  - `review`
    - 增加“角色当前能力是否支撑当前任务”的合理性提示
  - `consistency`
    - 增加“修为/能力层级 vs 场景风险/任务类型”的软校验
  - `worldbook`
    - 明确各修为阶段通常具备的能力、限制、常见任务边界
  - `story review`
    - 若正文出现明显越阶任务，应要求补足保护条件、制度解释或世界规则例外

## 3. 修炼阶段链条自相矛盾

- 现象:
  - 文本中出现：
    - “前世的顾长渊在金丹期才开始寻找归墟体”
    - “因为他需要归墟体来突破化神期的瓶颈”
  - 这两句并置时会让修炼阶段逻辑出现断裂。
- 影响:
  - 读者会质疑修炼体系时间线与阶段目标是否成立
  - 会削弱人物动机、修为 progression 与核心阴谋的可信度
- 初步归类:
  - 修炼体系阶段链冲突
  - 动机与境界目标错位
  - 世界规则/力量体系时间线不自洽
- 后续建议检查方向:
  - `consistency`
    - 增加“修炼阶段 -> 下一阶段目标”链条的软/硬校验
  - `worldbook`
    - 显式记录境界序列、各阶段常见瓶颈与突破条件
  - `review`
    - 当正文同时提到多个修炼阶段和突破目标时，检查它们是否构成合理 progression

## 4. 结构化方案文档口吻侵入正文

- 现象:
  - 正文中出现：
    - “沈玄深吸一口气，开始制定详细的行动方案。”
    - “时间窗口：三个月”
    - “目标：……”
    - “风险：……”
    - “约束：……”
  - 这类段落虽然整齐清楚，但更像项目方案、行动备忘录或现代计划文档，而不是玄幻叙事中自然流动的思维表达。
- 影响:
  - 容易让读者出戏，感到角色像在写 PRD / 任务计划
  - 会强化 AI 产物中常见的“结构化归纳腔”
- 初步归类:
  - 题材语域失真
  - 叙事呈现方式与角色时代感不匹配
  - 过度结构化的认知表达
- 后续建议检查方向:
  - `style`
    - 增加“方案文档化段落”检测，而不是只盯单个词
  - `review`
    - 判断这类段落是否被世界观合理化，例如角色确实受现代训练、前世职业相关
  - `genre-pack`
    - 玄幻题材默认弱化此类结构化标题句式，现代修仙/穿越工程师类题材可显式放宽

## 5. 章间承接与过渡断裂

- 现象:
  - 以《归墟》为例，第 12 章与第 13 章之间几乎没有自然过渡和承接。
  - 章节像是在各自完成细纲事项，但上一章结尾留下的情绪、信息、行动后果，并没有被下一章开头自然接住。
- 影响:
  - 读者会感觉剧情不连贯，像在读被拆开的提纲执行稿，而不是连续的小说。
  - 会削弱章节节奏、卷内推进感和长线阅读黏性。
- 初步归类:
  - 章节桥接质量不足
  - “贴合大纲/细纲”优先于自然叙事承接
  - workflow / review 尚未显式约束“上一章结果 -> 下一章起点”
- 后续建议检查方向:
  - `review`
    - 增加“是否自然承接上一章结果”的提示或评分维度
  - `workflow / context`
    - 在写前显式暴露“上章结果、本章承接点、本章必须交出的推进”
  - 人工审查
    - 把“章间衔接是否自然”列为固定必审项

## 6. AI 味过重但不只是一两个词

- 现象:
  - 部分正文虽然设定上没直接出错，但整体读感仍然“AI 味”过重。
  - 常见表现包括：
    - 程式化过渡句
    - 说明替代表演
    - 过于整齐的归纳结构
    - 为完成提纲事项而写，而不是为故事自然推进而写
- 影响:
  - 会削弱角色真实感、场景沉浸感和阅读连续性。
  - 即使局部规则通过，也可能整体可读性不足。
- 初步归类:
  - 风格问题
  - 叙事组织问题
  - 生成流程缺少写后修正闭环
- 后续建议检查方向:
  - `style`
    - 继续补强 AI 味检测，不局限于单词级禁忌词
  - `review`
    - 把“像提纲执行稿而非自然小说”显式转成优先动作
  - 工作流
    - 强化“检测 -> 自审 -> 修正 -> 复检”而非单次评分

## 7. 章间承接检测对真实修补不敏感

- 现象:
  - 在《命灯照骨》`chapter-023` 中，正文开头已补入：
    - 灰背仓断后余波
    - 梁上递线女人留下追兵与净页火
    - `灰背乙四` 线索承接
    - `@{沈照}` / `@{岳怀川}` 显式出场
  - 但 `review chapter` 仍持续命中 `chapterHandoffWeak`，并判定“当前章开头没有明显承接上一章《第二十二章 灰里递名》留下的状态或线索”。
- 影响:
  - 作者已经针对承接问题做了真实修补，系统却仍给出同样结论，会削弱对 review 信号的信任。
  - 这类低敏感度会让“先修文再复检”的闭环价值下降，逼作者退回人工判断。
- 初步归类:
  - `review` / `chapterHandoffSignals` 识别口径过窄
  - 承接检测更像关键词命中，而不是消费真实 handoff 语义
- 后续建议检查方向:
  - `review`
    - 放宽对开章承接的判断方式，不只盯固定命名或模板句
  - `context / review`
    - 更直接消费 `previousChapter.sceneGoals / beatHints / handoff summary`
  - 样例回归
    - 固定保留“已手动补承接但仍被判 weak”的案例，避免后续误以为检测已有效

## 8. 卷末动作章的张力识别仍偏失真

- 现象:
  - 在《命灯照骨》`chapter-023` 中，正文已经包含：
    - 收笔役正面阻拦
    - 第一笔认姓逼近
    - 净页火与追兵压近
    - 逃生窗口缩小
    - 沈照记忆代价落下
  - 但 `review chapter` 仍将其稳定评为：
    - `conflictTension = 6/20`
    - “冲突张力偏平，读者的追读理由还不够强”
- 影响:
  - 对卷末章和动作驱动章的评分参考价值不足。
  - 容易把“检测器没读到张力”误判成“正文本身没有张力”。
- 初步归类:
  - `review` 对动作阻力、时限压力、追兵逼近、代价余波的信号抽取不足
  - 评分链对中文短段、快节奏章法仍不敏感
- 后续建议检查方向:
  - `review`
    - 增加对时限词、追击态势、代价落点、对抗升级的综合信号识别
  - `consistency / analyzer`
    - 暴露更多“状态变化 / 代价变化 / 风险升级”候选供 review 消费
  - 样例回归
    - 固定保留卷末动作章样例，检查后续改动是否真的提升张力识别，而不是只换说法

## 9. 段落粒度不适配手机阅读

- 现象:
  - 当前证据主要集中在《命灯照骨》第 1 章，而不是后续章节的普遍问题。
  - 第 1 章在手机屏幕上展开后，会出现难以快速扫读的大段落。
  - 第 2 章起暂未观察到同等级问题，说明这更像首章局部失控，而不是全书统一段落策略失败。
- 影响:
  - 会直接降低首章的手机端阅读流畅度。
  - 由于问题出现在开篇，尤其容易影响首屏阅读体验与继续追读意愿。
- 为什么这轮没解决:
  - 生成阶段主要在追求首章氛围、信息建立与钩子，没有把“移动端段落长度”当成硬约束。
  - 自审阶段更关注承接、张力、AI 味和设定逻辑，没有单独检查首章在手机阅读视角下的段落负担。
  - 工具侧当前只有粗粒度的 `averageParagraphLength`，还没有把“移动端段落可读性”转成明确 judgement 或动作建议。
- 初步归类:
  - 表达控制问题
  - 连载平台阅读适配问题
  - `style / review` 对段落负担的检测维度不足
- 后续建议检查方向:
  - `style`
    - 增加面向移动端阅读的段落长度与连续信息密度提示，尤其优先检查首章与黄金三章，而不是只看全章平均值
  - `review`
    - 对商业连载项目显式增加“手机端可读性 / 段落负担”维度，并优先作用于开篇章节
  - 生成规则
    - 把“信息揭示段、回忆段、动作段都要控制单屏负担”纳入写中基本规则
  - 样例回归
    - 保留一批“桌面看还行、手机看显重”的开篇正文样例，避免后续只靠桌面读感误判

## 10. 基础设定与背景显性化不足，影响爽点与冲突感知

- 现象:
  - 在人工审查视角下，当前项目的基础设定、背景秩序和关键冲突底板仍不够显性。
  - 结果是读者不容易快速抓到：
    - 这个世界最核心的爽点引擎是什么
    - 主角当前被什么压着
    - 哪些压抑会在何处迎来反压制或爆发
- 影响:
  - 会削弱玄幻题材中“压抑 -> 蓄势 -> 反击 -> 爽点兑现”的阅读抓力。
  - 也会让后续章节即使有波折和代价，读者仍感觉缺少明确的背景支点与期待方向。
- 为什么这轮没解决:
  - 生成阶段更多依赖已有 project/worldbook/context，但没有强制把“世界底板 + 主爽点 + 当前压制源”反复显性化到卷章细纲与正文承接中。
  - 自审阶段更关注逻辑、承接、AI 味和局部评分，没有把“基础设定是否足够让读者读懂爽点引擎”列为固定问题。
  - 工具侧虽有 `project/worldbook/context`，但缺少面向作者审查的“设定底板摘要 + 当前卷爽点引擎摘要”产物。
- 初步归类:
  - 设定显性化不足
  - 背景压力与爽点引擎暴露不足
  - `outline / context / review` 对题材阅读预期支持不够
- 后续建议检查方向:
  - `outline / detailed outline`
    - 在卷级和章级都显式写出“背景底板、当前压制、预期爽点、爆发落点”
  - `review`
    - 增加“当前章/当前卷是否让读者清楚知道压抑与爆发的方向”这一类提示
  - 样例回归
    - 固定检查“读完开篇与卷首，是否已明白世界规则、当前困局和期待中的反压制”

## 11. “不是xx，是xx”类对照句式属于典型 AI 偏好，但当前未被稳定检出

- 现象:
  - 正文中频繁出现“不是……，是……”这类对照解释句式。
  - 这类句子单独看不一定错，但高频出现时会明显带出 AI 常见的说明腔和强调腔。
- 影响:
  - 会形成可感知的机械表达习惯。
  - 容易削弱叙事自然度，让正文更像在给读者“解释正确理解方式”。
- 为什么这轮没解决:
  - 生成阶段没有把这类句式当成高风险支架主动规避。
  - 自审阶段更关注宏观 AI 味、承接和逻辑，没有专门数这类高频句式。
  - 工具侧当前 `style` 更偏词表、支架复用、方案文档腔、语域漂移，没有把“不是A，是B”这类中文对照句式做成单独检测项。
- 初步归类:
  - AI 风格句式问题
  - 叙事支架复用问题
  - `style` 检测覆盖不足
- 后续建议检查方向:
  - `style`
    - 增加对“不是……，是……”“不是A，而是B”等高频对照句式的检测与阈值控制
  - 生成规则
    - 明确约束：少用解释性对照句替代直接呈现
  - 样例回归
    - 固定保留一批高频对照句案例，防止后续只抓词汇不抓句式

## 12. 势力、地点、特色物品缺少显式包裹引用，导致正文中的设定核对链不完整

- 现象:
  - 当前真实正文里，`@{}` 基本只稳定落在角色名上。
  - 但从人工审查需求看，真正需要快速核对的往往不止人物，还包括：
    - 势力
    - 地点
    - 特色物品
    - 特殊制度/岗位相关对象
- 影响:
  - 审查者很难快速确认正文里的关键对象是否已与 `worldbook / entities` 对齐。
  - 也会削弱后续“设定遵循、物品规则、势力行为”这类校验的抓手。
- 为什么这轮没解决:
  - 生成阶段没有把“关键非人物对象也要进入包裹引用链”当作稳定写中规则。
  - 自审阶段没有专门检查“本章高价值对象里，哪些仍停留在裸文本”。
  - 工具侧当前主要能校验“已包裹对象是否合法”，却还不能可靠发现“未包裹但应包裹”的对象。
- 初步归类:
  - 正文协议执行不足
  - 审查抓手不足
  - `doctor / review / export` 对非角色对象的消费链不完整
- 后续建议检查方向:
  - 写中规则
    - 明确哪些类型的对象默认应优先使用 `@{}` 包裹
  - `doctor / review`
    - 增加“关键对象应包裹未包裹”的提醒
  - 样例回归
    - 固定检查一章内是否只包角色、不包世界对象

## 13. 势力层级行为、地域扩张与战力扩张缺少稳定可审查约束

- 现象:
  - 在长篇玄幻语境下，读者会默认期待：
    - 中层势力有中层势力的做法
    - 小地方阶段与大地图阶段的资源、风险、人物盘面不同
    - 主角变强后，舞台、敌人层级、规则压力也会跟着扩大
  - 当前项目虽然已有基础设定与势力条目，但在正文审查时，仍难快速判断：
    - 某个宗门/组织的行为是否符合它宣称的层级
    - 当前篇幅是否还停留在旧地域、旧对手、旧战力盘面
    - 世界观是否随着主角成长自然扩宽
- 影响:
  - 容易出现“势力写得像空标签”“战力升级了但冲突层级没跟上”“世界观名义上很大、实际舞台始终没扩”的问题。
  - 这类问题若拖到很多章后才暴露，修复成本会很高。
- 为什么这轮没解决:
  - 生成阶段没有把“当前地域层级 / 当前战力上限 / 当前势力压制关系 / 下一步扩张方向”持续显性化。
  - 自审阶段没有固定检查“本卷的世界尺度与主角阶段是否匹配”。
  - 工具侧当前缺少把势力、地点、战力 progression 汇总成可读审查结论的能力。
- 初步归类:
  - 世界尺度控制问题
  - 势力层级逻辑问题
  - 长篇成长型题材的阶段审查不足
- 后续建议检查方向:
  - 卷级细纲
    - 显式写出本卷舞台范围、主要势力层级、当前战力天花板与下一步扩圈方向
  - `review / consistency`
    - 增加“势力行为是否符合层级”“当前冲突是否匹配主角阶段”“世界舞台是否自然扩张”的审查项
  - 样例回归
    - 固定保留“地方篇 -> 区域篇 -> 更大盘面”的成长链路样例，检查工具是否能看出扩张失衡

## 14. 前三章未讲清基础设定，原创命灯体系缺少足够的读者引导

## 15. 宗门压榨体系的因果链未立住，导致高危劳动与角色选择显得不合理

## 16. 反派与压迫方缺少足够行为逻辑和人格支撑，容易沦为功能性对立面

- 初步归类:
  - 反派塑造不足
  - 冲突对手逻辑不足
  - 人物驱动与功能驱动失衡
- 后续建议检查方向:
  - `review`
    - 增加“敌对角色是否具备自洽目标、行为逻辑、约束和人格辨识度”的审查项
  - 卷级/章级细纲
    - 为关键反派或压迫方显式写出其目标、手段边界、短期利益与长期信念
  - 样例回归
    - 固定检查反派是否只是制造麻烦，还是已经具备可理解的行动逻辑与人物魅力

## 17. 章节层级元信息泄漏进正文，破坏叙事次元与人物视角边界

- 现象:

- 影响:
  - 会立刻破坏沉浸感。
  - 会让读者感到提纲/章节管理视角直接穿透进了正文，属于高显眼度瑕疵，而不是普通修辞问题。
- 为什么这轮没解决:
  - 生成阶段可能混入了章节规划、回顾摘要或外层工作流语言，导致“章级元信息”错误落入人物视角叙述。

  - 生成规则
    - 明确禁止把章节编号、卷编号、提纲语汇、复盘语汇直接写进正常叙事正文
  - 样例回归
    - 固定保留这类“第三章里……”案例，作为高优先级硬伤样例

## 18. 抽象递进与故作深沉句式 AI 味过重，但当前未被稳定检出

- 现象:
  - 正文里会出现类似：
    - “心里竟没先起火。先起的是更清楚的一层算。”
  - 这类句子往往不是正常、具体、可感知的人物反应，而是把情绪或判断硬拧成一层抽象概念，制造一种“似乎很有意味”的腔调。
- 影响:
  - 会明显带出 AI 常见的伪深刻感和造句痕迹。
  - 读者未必会逐句指出问题，但会直觉觉得“不像人会这么想、这么说”，从而削弱人物真实度。
- 为什么这轮没解决:
  - 生成阶段没有充分约束“人物即时反应要具体、可感、贴近处境”，导致模型容易滑向抽象修辞堆叠。
  - 自审阶段更容易检查明显的词表问题和结构问题，但不容易抓到这种单句层面的“抽象腔”。
  - 工具侧当前虽已覆盖部分 AI 句式与方案文档腔，但还没有把“先起的是……”“更清楚的一层……”这类抽象递进表达视为高风险信号。
- 初步归类:
  - AI 风格句式问题
  - 人物反应抽象化过度
  - 故作深沉表达问题
- 后续建议检查方向:
  - `style`
    - 增加对抽象递进、伪深刻、非具体化人物反应句式的检测
  - 生成规则
    - 优先要求人物反应落到动作、感官、明确判断，而不是抽象概念层层叠加
  - 样例回归
    - 固定保留这类“先起的是更清楚的一层算”案例，作为高优先级 AI 味样例

## 19. `不是……是……`、`像……`、短句递进等支架句式复用过频，会形成明显 AI 腔

- 现象:
  - 正文中会连续出现这类句式组合：
    - “不是热。是紧。”
    - “不是临时撞上。是有人一开始就在挑。”
    - “不是……不是……是……”这类连续否定后再落判断的句式
    - “像有一根旧线……”
    - “不重。却……”
    - “不轻、不重、不偏、不倚”这一类刻意校准力度或姿态的短句
    - “真正值钱的，从来都是……”
    - “真正重要的/有价值的/该做的，是……”
    - “还有什么？”这类故意留半句、继续追问的短对话
  - 单句看并非绝对错误，但当 `不是……是……`、`像……`、短促断句递进反复叠加时，读感会迅速变得机械。
  - 其中一部分对话句式确实可能来自人物设定，例如角色习惯追问、说话留半截。
  - 但如果没有明显区分不同人物口吻，而是多个角色都频繁使用，就会从“人物习惯”滑成统一的 AI 对话支架。
  - 这不是一两个固定句式的问题，而是同一类“翻转、校准、递进、半截追问”的表达习惯被模型反复调用，最终形成统一腔调。
- 影响:
  - 会让文字像在按模板制造力度，而不是自然形成力度。
  - 读者会明显感觉到句子“都长一个味道”，从而削弱人物感受的真实度和文本的新鲜度。
- 为什么这轮没解决:
  - 生成阶段默认把这类句式当作“强化感受”和“制造节奏”的安全手段，缺少频率约束。
  - 自审阶段虽然能看到单句不对劲，但没有稳定量化“这一组支架句式在本章/本段是否已经过密”。
  - 工具侧当前对 `不是A，是B` 有单点问题意识，但还没有把它和 `不是……不是……是……`、`像……`、短句递进、`不重/不轻/不偏/不倚`、`真正……的，从来都是……`、`还有什么？` 这类追问短句一起视为一组高频 AI 支架来统查。
- 初步归类:
  - AI 风格句式问题
  - 支架句复用过频
  - 局部节奏模板化
- 后续建议检查方向:
  - `style`
    - 增加对 `不是……是……`、`不是……不是……是……`、高频 `像……` 比喻、短句递进连发、`不重/不轻/不偏/不倚` 这类校准句、`真正……的，从来都是……` 这类价值判断定句、`还有什么？` 这类追问短句的组合式检测，而不是只抓单一模式
  - 生成规则
    - 明确限制这类支架句的连续使用频率，要求更多回到具体动作、触感和环境反馈；若某类短对话被视为人物口吻，也应只在该人物身上稳定出现，而不是全员共用
  - 样例回归
    - 固定保留“不是热。是紧。像……”“像……不重。却……”“真正……的，从来都是……”和“还有什么？”这类组合案例，作为高频句式复用样例
  - 记录策略
    - 后续同类问题不再按单句逐条新增，而是统一并入这类“高频支架句风格习惯”样例池

## 20. 第一卷前段伏笔、势力、人物与特殊物品堆叠过密，回收不足，超出引入篇承载能力

- 现象:
  - 从第 1 章到第 10 章，疑云和伏笔几乎持续叠加，但已经形成完整回收的点很少。
  - 同时登场的势力、人物、特殊物品偏多，导致读者在开篇阶段就要同时记大量对象与关系。

- 影响:
  - 会削弱第一卷作为引入篇应有的清晰度和牵引力。
  - 读者容易在尚未建立稳定理解时就被迫处理过多悬念和名词，进而失去跟读意愿。

  - 生成阶段更倾向于持续加深神秘感和暗线密度，却没有同步控制“首卷读者的认知负担上限”。
  - 自审阶段没有稳定检查“本卷当前新增了多少未回收疑点、多少新势力/新物件、是否已经超过引入篇阈值”。

- 初步归类:
  - 卷级复杂度失控
  - 引入篇信息负担过重
  - 伏笔与兑现节奏失衡
- 后续建议检查方向:
  - `review`

  - 卷级细纲
    - 显式控制第一卷应重点建立的核心问题、核心势力、核心人物与核心物件数量
  - 样例回归
    - 固定检查首卷前半段是否存在“持续加谜、不够回收、对象过多”导致的理解过载

## 21. 角色视角可见性越界，正文写出了当前 POV 不可能直接知道的信息

    - “正落在沈照右后方一条废弃排灰沟的边沿。”

    - 全知视角
    - 镜头外客观叙述

  - 但当前语境并没有明确切换，因此会形成 POV 逻辑断裂。

  - 会破坏人物视角的可信度。
  - 读者会感觉叙述者忽然“飘到天上”看 scene block，而不是老老实实待在当前角色感知里。
- 为什么这轮没解决:
  - 生成阶段更容易优先追求画面调度和悬念落点，没有严格约束“这条信息当前是谁能看到的”。
  - 自审阶段没有把“可见性 / 可知性 / 方位感知是否属于当前 POV”作为高优先级检查项。

  - 视角一致性问题
  - 可知性越界问题
  - 镜头调度压过人物感知
- 后续建议检查方向:

  - 生成规则
    - 明确限制：紧贴人物视角时，只能写该人物能直接感知、合理推断或事后得知的信息
  - 样例回归
    - 固定保留“右后方落灰”这类案例，作为 POV 越界样例

## 22. `灯` 与 `人` 的底层关系到第 13 章仍未讲明，核心概念隐晦过度

- 现象:

  - 这说明作品最核心的底层设定仍停留在“有神秘感”，还没有转化成“读者能抓住的基本理解框架”。
- 影响:
  - 会让读者长期处在“知道这东西重要，但不知道它本质上是什么”的状态。
  - 一旦核心概念本身都没有立住，后续围绕它展开的危险、压迫、修炼、伏笔和反转就很难真正产生力度。
- 为什么这轮没解决:
  - 生成阶段更倾向于持续保留神秘感，没有把“核心概念至少要让读者知道到什么程度”做成明确阈值。
  - 自审阶段没有单独检查“到某一章为止，读者是否已经理解世界最关键概念的最低定义”。

- 初步归类:
  - 核心概念 onboarding 失败
  - 原创设定隐晦过度
  - 设定神秘感压过可理解性
- 后续建议检查方向:
  - `review`
    - 增加“到当前章节为止，核心概念是否已经具备最低可理解定义”的审查项
  - 卷级/章级细纲
    - 显式写出“灯与人的关系”最晚应在何章前让读者理解到什么程度
  - 样例回归
    - 固定检查原创体系作品在首卷中段前，是否已经让读者抓住核心概念框架

## 23. 为制造疑问而持续加谜，导致首卷引入篇更像过载的悬疑铺陈而非稳定带入

- 现象:
  - 当前文本存在“为了增加读者疑问而继续加问题”的倾向。
  - 伏笔、疑点和遮蔽层层叠加，但阶段性解答和理解锚点不足，导致读者不一定是在“被吊着想看下去”，也可能是在“越看越不懂”。
  - 这类写法更接近高密度悬疑解密的铺法，但当前题材与阅读承诺并不主要建立在纯悬疑解谜上。
- 影响:
  - 会削弱第一卷作为引入卷的功能。
  - 读者在尚未建立基本世界理解和情绪站位前，就被要求处理过多未解问题，弃读风险会明显上升。
- 为什么这轮没解决:
  - 生成阶段把“持续制造疑云”误当成了稳妥的追读手段，却没有同步设计足够的短期回收和解释节点。
  - 自审阶段缺少“这一卷是在勾人，还是已经让人看不懂”的复杂度判断。
  - 工具侧当前虽有伏笔和复杂度问题意识，但还没有专门判断“当前疑点密度是否已经偏向错误题材策略”。
- 初步归类:
  - 疑点堆叠失衡
  - 首卷题材策略偏移
  - 神秘感与可读性失衡
- 后续建议检查方向:
  - `review`
    - 增加“首卷疑点密度是否超过当前题材承载能力”“阶段性解答是否足够”的审查项
  - 卷级细纲
    - 明确哪些谜题必须留到中段/高潮，哪些核心疑问必须在引入卷内尽早解释
  - 样例回归
    - 固定检查首卷是否出现“不是悬疑文，却用了悬疑过载铺法”的问题

## 24. 当前 23 章不足以构成第一卷的小故事闭环，高潮有了但收尾未成立

- 现象:
  - 从当前阅读结果看，`chapter-017` 之后已经明显进入高潮推进，追兵紧逼、局势升级、节奏也开始真正起伏。
  - 但到 `chapter-023` 为止，仍难称之为一个完整小故事的收尾。
  - 也就是说，局部高潮成立了，但“第一卷作为独立单元应完成一次相对完整的起承转合与阶段性交付”并未成立。
- 影响:
  - 会削弱卷级阅读满足感。
  - 读者可能愿意承认过程紧张，却不一定愿意认可“这一卷值得单独追完”。
- 为什么这轮没解决:
  - 生成阶段更偏向持续抬升局势，没有同步围绕“这一卷要收束哪条主问题、交出哪次阶段性结果”做闭环设计。
  - 自审阶段此前更多停留在单章与局部承接，没有稳定从“卷=一个小故事”去审收尾是否成立。
  - 工具侧当前虽已有卷级自审方向，但还没有把“卷级闭环是否真正完成”变成硬门槛。
- 初步归类:
  - 卷级闭环失败
  - 高潮存在但收尾不足
  - 小故事单元未完成
- 后续建议检查方向:
  - `review`
    - 增加“本卷是否完成一个相对独立的小故事闭环”的审查项
  - 卷级细纲
    - 显式写出本卷主问题、阶段性高潮、收尾交付与卷尾余波
  - 样例回归
    - 固定检查“高潮成立但卷尾未闭环”的案例，避免以后误把紧张感当成完成度

## 25. 第一卷整体压抑过重，缺少明确爆发与爽点兑现，商业承诺不清

- 现象:
  - 当前整卷整体情绪长期处于压抑、受逼迫、被追赶的状态。
  - 虽然“欲扬先抑”是常见写法，但当前稿件几乎一直在“抑”，缺少足够明确、足够可感知的“扬”。
  - 结果是读者很难回答：
    - 这本书的爽点是什么
    - 读者为什么要继续追
    - 它主要提供怎样的阅读体验
    - 它面向什么人群
- 影响:
  - 会直接削弱商业连载中的追读理由。
  - 如果连首卷都无法让读者感受到至少一次明确的情绪回报或阅读承诺，弃读风险会明显升高。
- 为什么这轮没解决:
  - 生成阶段更专注于压迫感、危险感和谜团升级，没有同步安排足够清晰的阶段性反压制、兑现或快感时刻。
  - 自审阶段没有把“这一卷的爽点是否已经兑现过至少一次”列为稳定问题。
  - 工具侧当前对 hook、张力、承接有一定消费，但对“爽点引擎是否成立、是否已兑现”还不够强。
- 初步归类:
  - 情绪曲线失衡
  - 爽点承诺不清
  - 商业定位与体验承诺不足
- 后续建议检查方向:
  - `review`
    - 增加“本卷主爽点是否清晰”“是否已有阶段性爆发/兑现”的审查项
  - `project / PRD / 卷级细纲`
    - 显式写出目标人群、核心体验、主要爽点、压抑与爆发的节奏分布
  - 样例回归
    - 固定检查首卷是否只会压、不知道何时扬

## 26. 第一卷缺少清晰的引入卷结构意识，世界观、钩子与单元剧情没有稳定按经典节奏展开

- 现象:
  - 当前首卷没有稳定体现出“先立世界基本规则与主处境，再用黄金开篇抓兴趣，再展开一个相对完整的单元故事”的结构意识。
  - 从阅读体验看，更像持续铺谜与持续压迫，而不是按某种成熟的卷级结构在稳步带读者进入故事。
- 影响:
  - 会让首卷既不像纯悬疑，也不像成熟商业网文的引入卷。
  - 读者难以快速抓住“我现在该期待什么、这个卷到底要讲完什么”。
- 为什么这轮没解决:
  - 生成阶段没有把三幕式、五幕式或等价的卷级结构拆成明确的执行框架。
  - 自审阶段缺少“这一卷是否符合引入卷应有结构职责”的固定核查。
  - 工具侧也没有提供卷级结构模板或结构审查器来帮助 AI 对照检查。
- 初步归类:
  - 卷级结构框架缺失
  - 引入卷职责不清
  - 经典叙事结构未被工作流显式消费
- 后续建议检查方向:
  - 卷级细纲
    - 显式支持三幕式、五幕式或等价结构模板
  - `review`
    - 增加“引入卷是否完成世界观奠基、兴趣钩子、小单元剧情启动”的审查项
  - 样例回归
    - 固定检查首卷是否只是材料堆积，而不是按结构方法完成带入与交付
