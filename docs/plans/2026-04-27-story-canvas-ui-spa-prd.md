# Story Canvas Early UI SPA PRD

> 最后更新: 2026-04-27
> 状态: 开发中
> 关联当前入口: `docs/roadmap.md`

## 1. 背景

当前 `Story Canvas` UI 已有 `Vue 3 + Vite + TDesign` 骨架，但视觉语言仍偏宣传页和实验稿：圆角偏大、说明文字偏多、三页风格不够统一。与此同时，路线图已经把早期 UI 定义为“降低测试和审查成本的本地工作台”，不是独立品牌站。

因此需要先收敛统一视觉规则，再继续推进功能面板，确保后续 UI 扩展不再边做边漂。

## 2. 产品定义

| 字段 | 内容 |
|------|------|
| 产品 | Story Canvas Early UI |
| 形态 | 单页应用（SPA） |
| 技术栈 | Vue 3 + Vite + TDesign Vue Next |
| 当前定位 | 本地工作台 / 协议可视化壳 |
| 目标用户 | 本地使用 CLI + 协议文件进行小说测试与审查的操作者 |

## 3. 目标

1. 把 UI 从“装饰性骨架”收敛成“TDesign 风格的本地工作台”。
2. 保持单页应用结构，稳定三块主工作区：章节与审查、插画工作台、全局设置页。
3. 在不新增项目协议真相源的前提下，提高可读性、操作密度和生图前置配置可见性。

## 4. 非目标

- 不做独立营销官网
- 不做富文本正文编辑器
- 不在前端持久化新的项目状态
- 不把敏感 secret 写入项目协议

## 5. 用户问题

### 5.1 当前问题

- 页面装饰性过强，和 `TDesign` 默认工作台气质不一致
- 页面里存在过多副标题和大块说明文案，扫读成本高
- 各页信息组织方式不统一，后续继续扩展 review/context/change request 容易失控
- 生图前缺少对 API key / provider 配置状态的显式反馈，失败原因要等到操作时报错

### 5.2 目标体验

- 进入页面后能立刻看到项目状态、章节问题、插画历史
- 视觉上更接近后台工作台，而不是 landing page
- 新增 review/context/change-request 面板时能沿用同一套结构和样式规则
- 插画工作区在缺少 key 或 provider 时会提前告警，并能直接进入设置页完成配置

## 6. 功能范围

### 6.1 In Scope

- SPA 总体壳层收敛
- 概览页重排
- 章节与审查页继续补全为工作台
- 插画工作台风格收敛
- 设置页与工作台级配置接入
- 统一样式 token、圆角、层级、说明文字策略
- 引入 `ui/design.md` 作为视觉规范入口

### 6.2 Out of Scope

- 真实生成任务轮询
- review packet 文件级导出
- 富编辑能力

## 7. 设计要求

- 遵循 `ui/design.md`
- 默认圆角 `6px` 到 `8px`
- 除页面主标题外，减少 decorative subtitle / eyebrow / slogan
- 优先复用 TDesign 组件原生信息层级

## 8. 信息架构

### 8.1 概览

- 项目状态
- 章节统计
- workflow 状态
- 最近插画记录

### 8.2 章节与审查

- 章节列表
- 当前章节审查摘要
- 问题 / 动作队列
- 协议上下文片段

### 8.3 插画工作台

- 目标选择
- 模式与参数表单
- dry-run / generate 相关操作区
- 历史记录与结果预览
- 配置缺失告警与设置页跳转

### 8.4 设置页

- 本地 OpenAI API key 配置
- 当前项目 provider / model / base URL / 默认尺寸 / 质量配置
- 明确区分“工作台本地 secret”和“项目协议默认参数”

## 9. 验收标准

1. UI 保持单页应用结构，不拆成多应用。
2. 审查、插画、设置三块主页面视觉规则统一，明显减少大圆角和副标题文本。
3. 不新增前端真相源；新增的本地 API 只能持久化工作台私有 secret 和现有项目插画配置。
4. `npm run build` 通过。

## 10. 风险

- 仅做前端收敛时，部分“待接入”区域仍只能展示派生信息
- 若后续继续添加面板但不遵循 `ui/design.md`，容易再次漂移
