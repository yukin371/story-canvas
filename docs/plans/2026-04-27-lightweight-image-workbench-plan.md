# 轻量生图工作台与提示词系统实施设计计划

> 创建日期: 2026-04-27
> 状态: 草稿
> 关联当前入口: `docs/roadmap.md`

## 一、背景

生图能力已经具备最小 provider 基础，但产品层还没有完成“低门槛工作台化”。本计划用于把“模板驱动 + 多模式 + 未来可拆在线工具”的设计方向拆成可执行阶段。

## 二、目标

1. 把生图工作台收敛成轻量产品，而不是参数面板集合
2. 建立模板优先的提示词系统
3. 明确 Story Canvas 内嵌态与未来独立在线工具的能力分层

## 三、非目标

- 不在本轮做 ComfyUI 式节点系统
- 不在本轮做复杂图像编辑器
- 不在本轮做完整商业后台

## 四、范围

### 4.1 In Scope

- 生图工作台产品信息架构
- 提示词模板系统
- 文生图 / 图生图 / 批量 / 重绘的统一交互定义
- 普通版 / 商业版的前瞻分层

### 4.2 Out of Scope

- 具体 provider 接口扩展
- 新模型接入细节
- 大规模素材管理

## 五、前置假设

- 当前 provider 层会继续维持“底层能力稳定、产品层轻封装”的方向
- 模板系统未来既可被 Story Canvas 使用，也可被独立在线工具复用

## 六、设计结论

### 6.1 产品结构

- 一层：通用生图内核
  - 生成模式
  - 模板系统
  - 批量/重绘
  - 历史与重跑
- 二层：Story Canvas 场景增强
  - 章节默认 prompt
  - 角色默认 prompt
  - 项目默认配置

### 6.2 提示词结构

- `template`
- `style modifiers`
- `user extra prompt`
- `negative/commercial policy`

### 6.3 复杂度策略

- 默认轻量
- 高级设置折叠
- 模板优先于空白 prompt

## 七、实施计划

### Checkpoint A: 文档与协议口径

- [ ] 定义 prompt pack 的最小字段结构
- [ ] 定义模板分类体系
- [ ] 定义“普通版 / 商业版”预留字段边界

### Checkpoint B: Story Canvas 内嵌工作台

- [ ] 在 UI 中增加模板选择区
- [ ] 支持“模板 + 用户额外提示词”组合
- [ ] 增加批量数量与重绘入口
- [ ] 明确高级设置折叠区

### Checkpoint C: 历史与复用

- [ ] 支持历史任务重跑
- [ ] 支持模板与用户提示词回填
- [ ] 明确哪些字段应进入未来独立工具的通用层

### Checkpoint D: 独立化预研

- [ ] 抽离 Story Canvas 特有字段与通用字段
- [ ] 评估普通版与商业版模板包差异
- [ ] 评估未来在线服务的账户、计费、合规需求

## 八、最低验收标准

- 文档明确模板优先策略
- 文档明确四类基础能力
- 文档明确未来独立化方向

## 九、一般验收标准

- 后续实现可直接按本计划拆阶段推进
- Story Canvas 内嵌态与独立态不会互相冲突
- 普通版 / 商业版分层不需要推倒重来

- 文档一致性：与 `docs/roadmap.md`、`ui/design.md`、既有 illustration 计划不冲突
- 实现验证：TBD，待进入编码阶段

## 十一、风险与回滚

### 风险

- 模板体系过浅，无法真正降低使用门槛
- 模板体系过深，产品反而变重
- 商业版需求若过早侵入，会拖慢内嵌工作台落地

### 回滚路径

- 本轮仅为文档设计，可单独回退相关计划文档和 `ui/design.md`
