# 写作规则包

> 最后更新: 2026-04-27
> 适用对象: 作者 / 外部 AI / 审查代理
> 关联流程: [创作流程指南](./creative-workflow.md)

## 1. 目的

把“要怎么写、不要怎么写、写完后检查什么”收敛成仓库内的 canonical 规则入口。

这些规则服务三个阶段：

- 写前知晓规则
- 写中遵循基本规则
- 写后接受检测、自审、修正与复检

## 2. 总原则

1. 先服从故事真相，再服从细纲执行路径。
2. 先保证读者能读懂，再追求神秘感和暗线密度。
3. 先让人物像人，再让情节推动人物。
4. 先交付一个章节或一卷该交付的内容，再继续埋下一层问题。
5. 规则不是替代创作，而是限制明显失真、失衡和 AI 习惯。
6. 面向中文用户和中文正文时，默认用中文表达写作规则、提示词正文、审查意见和改写建议；英文只保留给内部键名、代码字段和兼容映射。

## 3. 写前必须明确

开始写某章前，至少要能回答：

1. 上一章实际交出了什么后果。
2. 这一章从哪里接。
3. 这一章最晚要交出什么推进。
4. 当前卷要完成什么小故事职责。
5. 当前章涉及哪些关键设定、关键实体、关键伏笔。
6. 本章哪些内容不能写偏：
   - 世界规则
   - 人物关系
   - POV 边界
   - 当前题材承诺

如果这些问题回答不出来，不应直接写正文。

## 4. 开篇与首卷规则

### 4.1 开篇规则

1. 前几章必须让读者理解主角当前处境，不可只给氛围不给处境逻辑。
2. 原创核心概念必须尽早给出最低可理解定义，不可一直只保留神秘感。
3. 开篇压迫体系至少要能回答：
   - 为什么主角会被压在这里
   - 为什么制度能运转
   - 为什么角色不能轻易脱身
4. 黄金开篇要先建立继续读下去的理由，而不是先堆满未解释疑点。
5. 手机端阅读优先控制段落负担，尤其首章不要长段堆信息。

### 4.2 首卷规则

1. 第一卷是引入卷，也是一个小故事单元，不是纯铺垫桶。
2. 首卷必须有：
   - 主问题
   - 阶段升级
   - 至少一次体验兑现
   - 阶段性收束
3. 首卷要控制复杂度预算：
   - 新人物、新势力、新物件、新谜题不要持续只增不收
   - 关键疑点需要有阶段性解释节点
4. 若题材不是悬疑主驱动，不要用“持续加谜”替代叙事推进。
5. 高潮不等于卷完成；若只有局势抬升，没有交付感，首卷仍未闭环。

## 5. 结构与承接规则

1. 章节不是提纲事项的直译稿。
2. 每章都应至少完成一项：
   - 推进局势
   - 改变关系
   - 暴露代价
   - 回收信息
   - 放大冲突
3. 下一章开头必须接住上一章留下的结果、情绪或行动，不可像重开新场。
4. 如果当前章只是重复上一章的压力，不应视为有效推进。
5. 卷内应有压抑与爆发的分布，不可长期只压不扬。

## 6. 角色与对手规则

### 6.1 主角与主要角色

1. 角色行为先服从其处境、认知、欲望和代价。
2. 不要为了贴细纲，让角色突然像另一个人。
3. 人物成长可以推动剧情偏移，但不能变成无目标漂移。
4. 高频出场角色要有变化轨迹：
   - 立场变化
   - 关系变化
   - 状态变化
   - 认知变化

### 6.2 反派与压迫方

1. 反派不能只承担“制造麻烦”的功能。
2. 对手至少要讲清：
   - 目标
   - 利益
   - 约束
   - 做法
3. 反派行为需要符合其层级、资源和人格，而不是单纯发蠢配合主角。
4. 压迫系统要让读者看懂其效率和残酷来源，而不是只看见坏。

## 7. 设定、实体与伏笔规则

### 7.1 设定规则

1. 原创设定应先让读者抓住用途、代价、危险边界，再逐步扩深层解释。
2. 世界尺度、势力层级、战力阶段要和当前卷进度匹配。
3. 主角成长后，舞台、敌人和规则压力也应自然扩张。

### 7.2 实体规则

1. 关键人物、势力、地点、物品、制度对象，优先进入可追踪引用链。
2. 正常叙述里，关键 canon 实体应优先使用 `@{实体}`。
3. 引号内称呼、口头别名、情绪性称呼通常不强制包裹。
4. 如果正文出现高价值实体却未进入引用链，应视为待补档或待包裹对象。

### 7.3 伏笔规则

1. 伏笔分短、中、长期，不能一律只记账不回收。
2. 首卷不应只种钩子不收短线回报。
3. 每卷至少要回收一部分前置承诺，避免读者长期悬空。

## 8. POV 与叙述边界规则

1. 紧贴人物 POV 时，只写该人物能直接感知、合理推断或事后确认的信息。
2. 不要写当前 POV 无法直接看到的空间细节或敌方动作。
3. 不要把章节、卷、提纲、流程等作者侧元信息写进正文。
4. 若项目没有明确第四面墙玩法，默认禁止 meta 叙述。

## 9. 风格与 AI 风险规则

### 9.0 中文创作提示语言规则

1. 中文小说创作默认使用中文 prompt 正文。
2. 中文小说审查、重写要求、示例段落默认使用中文。
3. 若系统内部需要英文键名，如 `sceneEngine`、`firstVolumePromise`，应视为程序字段，不是给模型的正文指令语言。
4. 不要把英文概念短语直接拼接成中文创作提示，这很容易诱发翻译腔和英文推理骨架。

### 9.1 总体要求

1. 优先具体动作、感官和明确判断。
2. 少做抽象递进和伪深刻包装。
3. 少用解释性翻转句替代表演。
4. 避免整章统一腔调。

### 9.2 高频风险习惯

以下表达不是绝对禁用，但不能高频统治正文：

1. `不是……是……`
2. `不是……不是……是……`
3. `不是……更像……`
4. 高频 `像……`
5. `不重/不轻/不偏/不倚`
6. `真正……的，从来都是……`
7. `还有什么？`
8. 抽象递进句
9. 故作深沉反应句
10. 方案文档腔和总结腔

### 9.3 替代方向

1. 用具体动作替代抽象判断。
2. 用物理反馈替代“像……”连发。
3. 用角色口吻差异替代模板化对话追问。
4. 用场景结果替代“不是 A，是 B”的解释落点。

## 10. 写中最小自查

写完一章初稿后，至少快速自查：

1. 这一章有没有真实推进，而不是只换一种说法重复压力。
2. 下一章开头是否有可承接的结果。
3. 有没有当前 POV 不可能知道的信息。
4. 有没有正文元信息泄漏。
5. 有没有同类 AI 支架句密集复用。
6. 有没有关键实体还停留在裸文本。
7. 有没有新加谜题却完全没有阶段解释或回报。

## 11. 写后修正原则

1. 优先修结构性问题，再修局部句子。
2. 先修读者会弃读的问题，再修措辞精致度。
3. 若细纲与人物/设定真相冲突，应先调整执行路径，再硬修正文。
4. 若一卷未闭环，不要因为单章分数尚可就提前交人工终审。

## 12. 与工具的关系

1. 本规则包是当前写作约束入口，不是新的协议真相源。
2. CLI 检测、评分、自审与导出应逐步消费这些规则。
3. 在 CLI 还没完全工具化之前，adapter 与人工审查都应先以此作为统一口径。
