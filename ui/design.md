# Story Canvas UI Design

> 最后更新: 2026-04-27
> 适用范围: `ui/`
> 状态: 当前统一视觉规则入口

## 1. 目标

- 让早期 UI 看起来像 `TDesign` 体系内的本地工作台，而不是独立品牌站
- 优先服务“项目切换、审查浏览、插画操作”三类高频任务
- 在单页应用前提下，保持结构稳定、信息密度可控、后续面板可继续扩展

## 2. 视觉原则

### 2.1 基调

- 偏产品工作台，不做宣传页
- 以中性色 + 单一暖色强调为主，不依赖大面积装饰性渐变
- 保留少量质感，但不使用夸张品牌叙事文本

### 2.2 TDesign 对齐

- 优先使用 `TDesign` 默认布局、卡片、表单、表格、菜单、标签体系
- 页面层级主要靠间距、边框、标题和数据分组体现，而不是靠大段副标题
- 所有自定义样式都应尽量像是对 `TDesign` 的轻量收敛，而不是另起一套设计系统

### 2.3 圆角与阴影

- 默认圆角控制在 `6px` 到 `8px`
- 大卡片、列表项、按钮、输入框不使用 `12px+` 的圆角
- 阴影保持轻量，优先使用边框和底色分层

### 2.4 文本层级

- 页面只保留一个主标题
- 非必要不写 eyebrow / kicker / slogan 式副标题
- 说明文字只出现在用户真正需要决策或理解状态的位置
- 卡片标题尽量直接，例如“项目状态”“章节列表”“插画历史”

## 3. 布局规则

- 采用单页应用，左侧导航 + 顶部项目操作区 + 主内容区
- 主工作区必须占满可用高度，内部面板在 `100%` 高度下再做滚动分配
- 主内容区优先使用 2 栏以内布局，避免过度碎片化
- 桌面端强调快速扫描；窄屏自动退化为单列

## 4. 组件规则

### 4.1 导航

- 左侧导航只保留核心工作区入口
- 品牌区收缩为名称 + 简短状态，不放长文案
- 全局设置入口不与工作区入口混排，应作为左侧底部的独立图标入口

### 4.2 卡片

- 卡片内部优先容纳一类信息或一个动作组
- 少用“卡片套卡片”；若必须嵌套，内部容器应更轻

### 4.3 数据展示

- 统计数字、状态标签、更新时间优先用简洁的键值结构
- 问题列表、动作队列、协议字段优先用表格、列表或 description 风格组件

### 4.4 表单

- 插画工作台保持“目标 -> 模式 -> 参数 -> 操作”顺序
- 避免把提示文案写成大段设计说明，保留必要 placeholder 即可

### 4.5 生图工作台

- 默认路径应是“模板 -> 用户补充提示词 -> 生成”，不是让用户从空白 prompt 开始
- 若缺少 API key 或 provider 配置，必须在插画工作区给出显式告警，并提供进入设置页的入口
- 首屏优先展示：
  - 生成模式
  - 模板
  - 额外提示词
  - 数量 / 参考图 / mask
- 文生图、图生图、批量生图、重绘必须属于同一产品心智，不拆成彼此割裂的页面
- 高级参数默认折叠，不让新手一开始面对完整 provider 参数表
- 后续如果抽离成在线工具，当前布局应尽量保留通用生图产品的骨架，而不是写死为小说专用 UI

## 5. 当前阶段限制

- UI 不能发明新的持久化状态
- 若后端尚未提供真实数据，前端只能展示“待接入”或基于现有 summary 的只读派生信息
- 当前阶段不做富文本编辑器，不做复杂动效，不做多用户协作入口

## 6. 本轮收敛项

- 去掉现有页面中过多的大圆角和装饰性副标题
- 收敛为更克制的工作台布局
- 统一三页视觉语言，避免每页像不同产品
- 为后续“模板驱动生图工作台”预留轻量交互结构
