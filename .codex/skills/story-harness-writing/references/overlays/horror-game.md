# Horror-Game Overlay

适用定位：

1. `primaryGenre=thriller`
2. `subGenre=horror-game`
3. 常伴随 `rule-creepypasta`、`survival-loop`
4. 可兼容 `identity-mystery`、`cheat-variant`
5. 目标读者接近 `rule-horror-reader`

证据基础：

1. `noval-data/archetypes/horror-game/README.md`
2. `noval-data/analysis-workbench/synthesis/genre-overlays/horror-game.md`

## 当前高置信写法

1. 开场优先给副本规则压力和求生限制，不要先把惊悚写成单纯怪物袭击。
2. 主角的例外项要一边帮助破局，一边继续制造不安和未知代价。
3. 单章推进最好同时命中两条：
   - 规则理解前进
   - 身份谜团前进
4. 若带外挂变体，外挂更适合改写解法，而不是直接删除难度。
5. 惊悚感应来自规则边界、身份错位和未知后果，而不是只靠跳脸。

## 节奏建议

1. 首章先让读者知道“这套规则哪里会死人”，再给主角第一次异常反应。
2. 前几章至少要出现一次：主角发现某条规则未必适用于自己。
3. 第一卷更稳的交付，是主角从普通求生者升级到明确意识到自己不是普通玩家，或稳定掌握一类高风险破局方式。

## 常见误区

1. 只有通关流程，没有持续惊悚压迫。
2. 主角例外项太像纯外挂，失去不安感。
3. 规则很多，但没有逐层理解和回报。
4. 身份谜团和副本推进长期断开。
5. 外挂太万能，副本很快失去压迫感。

## 与普适层的分工

这层补的是：

1. 规则求生 loop
2. 例外项与身份谜团的双线推进
3. 副本惊悚感的来源控制

不要用它替代 `writing-universal.md` 或 `workflow-gates.md`。
