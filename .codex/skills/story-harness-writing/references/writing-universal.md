# Universal Writing Base

这是 `story-harness-writing` 的普适写作层。

在题材、平台、tag 尚未明确，或对应 overlay 尚未沉淀时，先只加载这一层。

## 目标

适用于大多数中文长篇 / 连载小说项目的基础写作规则：

1. 标题承诺
2. 主角异常态
3. 章级推进
4. scene engine
5. 卷级小闭环
6. 常见 AI 行文风险

## 基础规则

### 1. 标题承诺

章节标题、卷标题、作品定位都不是装饰。正文必须尽快兑现其承诺，而不是长期借标题吊着读者。

### 2. 主角异常态

主角不能只是“在做事”，还应处于一种值得继续观察的异常态：

1. 被压制
2. 被误认
3. 被追踪
4. 有代价
5. 知道的信息不完整
6. 与正常秩序有摩擦

### 3. 章级推进

每章至少要交付一种有效变化：

1. 局势推进
2. 信息推进
3. 关系推进
4. 状态推进
5. 代价推进

如果只是重复上一章的压力，不算有效推进。

### 4. Scene Engine

一个 scene / 场景至少要有：

1. 目标
2. 阻力
3. 变化

若只有解释、说明、设定介绍，而没有目标与阻力，这个场景通常会发空。

### 5. 卷级小闭环

一卷不是章节桶。它至少应提供：

1. 明确的卷问题
2. 阶段升级
3. 一次可感知交付
4. 局部 closure
5. 更大故事的尾巴

### 6. AI 风险

警惕这些基础风险：

1. 提纲直译成正文
2. 解释先于场面
3. 总结口吻压过动作和对白
4. 为过规则机械加桥接句
5. 句式簇重复过多

### 7. 语料信号转散文

语料分析和 overlay 不是命令清单。写正文前，先把它们压成一份内部风格简报：

1. `叙述距离`: 贴身、半贴身、旁观，决定能写哪些感知和判断
2. `行文速度`: 短促推进、轻快抛接、慢热沉浸，决定段落长度和信息密度
3. `句子质地`: 硬、冷、轻、潮湿、烟火气、少年感、古意，决定比喻和动词选择
4. `对白压力`: 试探、顶撞、遮掩、调情、审问、互相拆台，决定台词是否直说
5. `意象库`: 本章能反复压住情绪的物件、地点、声响、气味、伤痕
6. `禁用语域`: 本题材不该冒出来的方案文档腔、作者总结腔、翻译腔或过度网感

执行规则：

1. 不把上述字段写进正文。
2. 不把“要推进关系”写成解释句，而是写成一次越界、让步、误会、照顾、失控或沉默。
3. 不把“要交付代价”写成总结句，而是写成损坏的物件、变差的处境、失去的选择或别人改变的眼神。
4. 不把“要承接上一章”写成固定回顾，而是让上一章的物件、伤势、没说完的话、追兵、账目或情绪残留进入当前动作。
5. 不把“要有钩子”写成突然来人，而是让读者已经在意的东西出现新解释、新风险或新归属。

如果一段文字读起来像任务说明，先问：

1. 谁在现场承受这条规则？
2. 哪个物件、身体反应或关系边界发生了变化？
3. 哪句话可以改成动作、停顿、打断或反问？
4. 这条变化能否在下一段继续留下后果？

## 章节工艺

### 1. 前 20%

一章前 `20%` 基本决定读者会不会继续读。

优先交付：

1. 即时冲突或不稳定感
2. 重要信息或异常事件
3. 情绪牵引
4. 继续读下去的理由

开头常见死法：

1. 天气和日常流程先行
2. 复述上一章
3. 大段背景说明
4. 平淡寒暄
5. 解释代替动作

### 2. 结尾钩子

每章结尾最好留下一个明确追读理由，但不要机械地都写成“突然有人来了”。

高频有效钩子：

1. 新揭示改变现状理解
2. 迫近危机
3. 未完成动作
4. 身份或关系反转
5. 两难选择
6. 神秘物件或线索
7. 倒计时
8. 留白后的不安

禁区：

1. 虚假悬念
2. 低风险悬念
3. 同质化钩子连发
4. 一章只提新悬念，不回应旧悬念

### 3. 对话层次

对话不是填充字数。每句至少承担一种功能：

1. 推进情节
2. 揭示人物
3. 制造冲突
4. 传递新信息
5. 释放情绪
6. 制造悬念

额外要求：

1. 能短就短，不写论文腔台词
2. 尽量让角色不把心里话全部直说
3. 用动作、沉默、打断和转话题制造潜台词
4. 不要所有角色说话一个味道

### 4. 节奏与呼吸

节奏不是全程拉满，也不是全程匀速。

至少检查：

1. 长短句交替
2. 长短段交替
3. 高信息密度和低信息密度交替
4. 动作段落收紧，思考或消化段落放缓

若全章像一条直线，通常说明节奏死了。

### 5. 扩写原则

字数不够时，优先补“故事需要但写薄了”的部分，不要注水。

先诊断不足类型：

1. 场景太薄
2. 关键时刻跳太快
3. 情绪太平
4. 对话只传信息
5. 事件层太少

优先扩写方式：

1. 补场景肌理和感官维度
2. 放慢关键动作或决定时刻
3. 展开内心判断链
4. 让对话多一层潜台词
5. 穿插能服务主线的次要线索

判断标准：

1. 删掉这段，故事会不会变差
2. 它有没有推进人物、关系、线索或氛围
3. 它有没有破坏本章节奏

若答案偏负面，这段通常就是注水。

### 6. 连贯性

新章不是凭空开始。

写前至少要知道：

1. 上一章停在什么情绪和风险上
2. 当前章要回应哪条旧悬念
3. 当前章新增或升级哪条悬念
4. 关键人物此刻的位置、关系、已知信息、能力状态

若这些没对齐，就很容易写出“角色忘了自己知道什么”的问题。

## 人物工艺

### 1. 人物发动机

人物不能只靠“设定标签”活着，至少要有：

1. 想得到什么
2. 为什么非得到不可
3. 最怕失去什么
4. 一个会在关键时刻害他的缺陷
5. 一组和核心矛盾有关的内在冲突

高频有效矛盾：

1. 嘴硬但在意
2. 想保护人却不信任人
3. 追求秩序却靠越界手段活着
4. 最不信命却偷偷给自己留护身符

不要为反差而反差。矛盾必须能驱动选择和代价。

### 2. 用选择和习惯写人

不要直接贴“他很愤怒”“她很善良”这类标签。

优先用：

1. 压力下的选择
2. 重复出现的小习惯
3. 对弱者、下位者、陌生人的反应
4. 做完事后的余波动作

判断标准：

1. 删掉性格形容词，人物还立得住吗
2. 只看他的选择，能不能猜到他的价值排序

### 3. 缺陷必须会咬人

主角缺陷不能只写在档案里，要在关键节点制造失败或代价。

例如：

1. 傲慢导致误判
2. 逃避导致问题滚大
3. 信任障碍让盟友离开
4. 完美主义拖慢出手时机

如果缺陷从不造成损失，它通常只是装饰。

### 4. 关系要变化

关系不是静态名词，而是持续变化的张力。

至少要能回答：

1. 这一章前他们是什么关系
2. 这一章后近了还是远了
3. 这种变化是靠什么事件换来的
4. 变化有没有体现在说话方式、边界感、默契或敌意上

不要把关系推进只写成“想法变了”，要落到互动细节。

## 规划工艺

### 1. 从承诺到章节

大纲不是章节列表，而是承诺分配表。

从上到下至少要对齐：

1. 作品 promise
2. 卷问题
3. 当前章任务
4. scene 级目标与阻力

若一章不知道自己在卷里负责什么，它很容易沦为流水账。

### 2. 每章的最低规划粒度

在真正动笔前，至少应说清：

1. 本章核心事件
2. 承接上章什么落点
3. 本章要兑现什么
4. 本章结尾留下什么新问题
5. 哪些人物和线索必须出场

这五项说不清，正文大概率会散。

### 3. Scene 不是摘要切片

一个 scene 不该只对应“这里发生了什么”，还要对应“为什么必须写这个场景”。

scene 常见职责：

1. 推进局势
2. 逼人物做选择
3. 交付新信息
4. 改写关系
5. 抬高风险

若这个场景删掉后不影响以上任何一项，它通常还没找到职责。

术语边界：

1. `scene` 指章节内的场景 / 场次 / 功能段。
2. 三幕式、五幕式、英雄之旅里的“幕”应理解为 `act`，不要和 `scenePlans` 混用。
3. `review scene` 评的是局部场景功能，不是在判断传统 act 是否成立。

### 4. 结构模板只是骨架

三幕、三章、五章、英雄之旅之类结构可以当起步骨架，但不是强制真相。

更重要的是：

1. 起点和承诺是否清楚
2. 中段是否持续升级
3. 低谷是否真的逼出选择
4. 结尾是否既有交付又有余波

不要为了套结构牺牲题材节奏和项目协议。

## Hook 编排

### 1. Hook 不只在结尾

好的 hook 至少有三层：

1. 开头 hook：先拉住读者
2. 章节内 hook：场景和场景之间持续给推进理由
3. 结尾 hook：把读者推到下一章

如果只有结尾钩子，中段常常会塌。

### 2. 回应旧问题，再抬新问题

最稳的编排不是无穷加谜团，而是：

1. 回应一个旧问题
2. 升级一个已知风险
3. 再抛一个更难的新问题

读者需要同时得到回报和牵引。

### 3. Hook 要分层运行

至少区分三种长度：

1. 短弧：`1-3` 章内就该有回应
2. 中弧：一卷内逐步揭露
3. 长弧：跨卷或全书级核心问题

不要把所有东西都拖成长弧，否则读者只会觉得一直不兑现。

### 4. Hook 要换型

不要每章都靠同一种钩子续命。

可轮换的维度包括：

1. 信息揭示
2. 风险逼近
3. 关系翻转
4. 选择困境
5. 物件或异象
6. 留白后的不安

同型 hook 连发太久，会显得模板化。

## 使用方式

这层解决的是“怎么写得像小说”，不是“仓库流程如何闭环”。

要执行真实项目工作流时，再继续加载：

1. `workflow-gates.md`
